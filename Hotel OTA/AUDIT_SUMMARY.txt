================================================================================
BUGS FIXED IN THIS COMMIT
================================================================================

1. BUG-1 (CRITICAL): Hotel Staff Authorization Bypass
   Status: FIXED
   File: apps/api/src/middleware/auth.ts
   Issue: Hotel staff JWT token claims not verified in database
   Fix: Added prisma.hotelStaff lookup to verify staff member exists and 
        belongs to claimed hotel, preventing token forgery attacks
   Impact: Prevents unauthorized cross-hotel access

2. BUG-7 (HIGH): Inadequate Inventory Date Validation
   Status: FIXED
   File: apps/api/src/routes/booking.routes.ts
   Issue: No range validation on checkin/checkout dates
   Fix: Added zod validators for future-only dates, max 365 days ahead,
        and checkout > checkin
   Impact: Prevents inventory inconsistencies from past/future dates

3. BUG-16 (MEDIUM): Hardcoded CORS Fallback URLs
   Status: FIXED
   File: apps/api/src/index.ts
   Issue: Production has hardcoded CORS domains as fallback
   Fix: Removed fallbacks, require all origins via env vars with startup
        validation that fails if no origins configured in production
   Impact: Prevents use of outdated/compromised domains

4. BUG-18 (MEDIUM): Cancellation Penalty Boundary Condition
   Status: FIXED
   File: apps/api/src/services/booking.service.ts
   Issue: Off-by-one in "within 24 hours" check used > instead of >=
   Fix: Changed boundary from > to >= for consistent policy enforcement
   Impact: Clarifies policy: cancellations in 24h window incur penalty

5. BUG-21 (MEDIUM): Suspended Users Can Complete Bookings
   Status: FIXED
   File: apps/api/src/services/booking.service.ts
   Issue: No check for user.isActive before confirming booking
   Fix: Added user.isActive verification before confirmation
   Impact: Prevents suspended accounts from completing reservations

6. BUG-22 (MEDIUM): Inventory Negative Available Rooms
   Status: FIXED
   File: apps/api/src/routes/hotel-panel.routes.ts
   Issue: No validation that availableRooms >= 0 and <= totalRooms
   Fix: Added integer validation and capacity range checks
   Impact: Prevents invalid inventory states that break booking logic
